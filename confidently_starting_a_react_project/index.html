The Paradox of Choice

The Paradox of Choice - Why More Is Less is a 2004 book by American psychologist Barry Schwartz. In the book, Schwartz argues that eliminating consumer choices can greatly reduce anxiety for shoppers.

Autonomy and Freedom of choice are critical to our well being, and choice is critical to freedom and autonomy. Nonetheless, though modern Americans have more choice than any group of people ever has before, and thus, presumably, more freedom and autonomy, we don't seem to be benefiting from it psychologically.

 quoted from Ch.5, The Paradox of Choice, 2004

Project Setup

Per Pete Hunt, you are going to want to pick up ES6 along the way, so you want to be sure to support it in your build/bundle process, so ES6 (although you don't need ES6 in all-its-depth) will be a requirement of the boilerplate. Also, so we can avoid choosing between CommonJS and AMD, lets just learn ES6 Modules real fast:

With ES6, you have 2 syntaxes. You can either have multiple named exports or a single un-named export:

Use Multiple named exports for some library like Math where you want to provide multiple helpful routines.

//------ lib.js ------
export const sqrt = Math.sqrt;
export function square(x) {
    return x * x;
}
export function diag(x, y) {
    return sqrt(square(x) + square(y));
}

//------ main.js ------
import * as lib from 'lib';
console.log(lib.square(11)); // 121
console.log(lib.diag(4, 3)); // 5

//------ main.js ------
import { square, diag } from 'lib';
console.log(square(11)); // 121
console.log(diag(4, 3)); // 5
You can also import the complete module:

Use a default where it makes more sense for a single and un-needely named export.

For example, a function:

//------ myFunc.js ------
export default function () { ··· } // no semicolon!

//------ main1.js ------
import myFunc from 'myFunc';
myFunc();

Pet Hunt recommends Webpack, and I couldn't agree more. I think the choice about transpilers has really ended. The community has really settled on Babel 6, but in order for Babel to win they went through a very competitive period of innovation where it went from being a specific tool to convert ES6 to ES5 (aka 6to5), to a more general transpiler (Babel). To a highly configurable transpiler (Babel 6). We need to be sure our webpack setup is configured for Babel 6.
